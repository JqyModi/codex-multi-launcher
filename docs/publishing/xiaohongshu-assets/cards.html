真实痛点
每天重复几次，就会变成摩擦

不是不会配置，是不想每次都重来。

01
打开前先确认是哪套配置 官方 API、第三方中转、自建代理混在一起，很容易开错。
02
切项目之前先备份 config.toml 不是怕麻烦，是怕改完以后另一个窗口也被影响。
03
换 API Key 后担心串配置 多个项目并行时，最怕请求走到错误的 Base URL。
小问题重复出现，就是工作流成本 01
解决方式
让每个 Codex 窗口都有自己的配置

一个 Profile，就是一套独立工作区。

双击打开 每个 Profile 都生成自己的启动器 App，像普通应用一样打开。
不修改原 Codex App 02
当前能力
不是全能适配器，先说清边界

Key

URL
独立 Base URL 每个 Profile 可以指向不同服务商。
ID
模型可手动填写 /models 获取不到，也能继续手动输入。
API
