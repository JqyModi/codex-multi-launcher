Codex 多账号 · ChatGPT 多开

在不同窗口登录不同 ChatGPT 账号。

为工作账号、个人账号或不同订阅分别创建 Codex 配置。每个配置使用独立应用数据目录，不会因为切换一个窗口的账号而退出其他窗口。

多账号隔离如何工作

每个 Profile 都使用自己的 CODEX_HOME 和 --user-data-dir。账号登录状态、应用缓存、项目状态和 Profile 配置保存在各自目录中。

  • 官方 Codex / ChatGPT App 本身不会被替换。
  • 不同 Profile 不共用应用数据目录。
  • 账号凭据继续由官方 App 管理，工具不会读取或保存账号密码。
  • 删除某个 Profile 不会删除其他 Profile 或源 App 的登录状态。
隔离的是本地桌面应用状态。账号本身的订阅、额度和云端数据仍由对应 ChatGPT / OpenAI 账号决定。

创建账号登录配置

  1. 点击左侧配置列表旁的新增按钮。
  2. 填写便于识别的名称，例如“工作账号”或“个人 Plus”。
  3. 在登录方式中选择“ChatGPT 账号登录”。
  4. 按需选择是否同步已有历史对话。
  5. 生成并打开新配置，在独立窗口内完成登录。

账号登录与 API Key 的区别

方式适合场景需要填写
ChatGPT 账号Plus / Pro 等账号内使用在窗口内完成登录
API Key官方 API 或兼容服务商API Key、模型、接口地址

账号模式创建完成后，详情页不会显示接口地址和 API Key 输入框，避免用户误以为必须填写或意外覆盖登录方式。

常见问题

一个账号可以同时打开多个 Profile 吗？

可以尝试，但账号并发限制和服务条款由 OpenAI 决定。多开助手只负责本地目录隔离。

修改一个 Profile 的登录账号会影响其他 Profile 吗？

正常情况下不会，因为每个 Profile 使用不同的应用数据目录。

重启电脑后还需要重新登录吗？

登录状态应保存在对应 Profile 的应用数据目录中。若每次重启都丢失，请提交诊断信息和 Codex / ChatGPT 版本。
