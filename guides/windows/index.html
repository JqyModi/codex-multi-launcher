Codex Windows · WindowsApps

Windows 上的安装、创建与启动排错。

Windows 版本的主要差异来自 App 安装来源、WindowsApps 目录权限、符号链接和可执行文件路径。最新版已针对这些链路做兼容处理。

先确认 Codex / ChatGPT 安装来源

Windows 桌面 App 可能来自官网安装器，也可能以 Microsoft Store / AppX 包形式安装。两者最终文件位置和权限模型不同。

  • 普通安装：可执行文件通常位于用户可访问的程序目录。
  • Store / AppX：文件可能位于受保护的 C:\Program Files\WindowsApps。

最新版会优先使用用户在设置中指定的 App 路径；未指定时再从候选桌面 App 中自动检测。

创建 Profile 时出现 EPERM

典型错误是复制 Codex 插件缓存时无法创建 symlink。Windows 普通用户对符号链接和受保护目录的权限与 macOS 不同。

最新版在 Windows 同步插件缓存时采用 Windows 兼容处理，不要求沿用 macOS 的符号链接复制方式。
  1. 升级到 Latest Release。
  2. 关闭正在运行的旧版多开助手。
  3. 重新创建 Profile，不要复用创建失败留下的不完整目录。
  4. 若仍失败，确认安全软件没有阻止用户目录写入。

创建成功但打开失败

提示找不到 ChatGPT.exe

进入设置，手动选择正确的 Codex / ChatGPT App 路径。合并后的桌面 App 可执行文件名可能发生变化。

窗口一闪而过

先确认官方 App 本身可以正常启动，再检查 Profile 详情中的 App 路径。虚拟机环境还可能受到图形加速、会话退出或共享目录执行限制影响。

Store / WindowsApps 权限提示

不要手工修改 WindowsApps 权限。优先使用官网可直接安装的桌面版本，或在 App 设置中指定可正常启动的程序路径。

提交问题时请附带

  • Windows 版本与 CPU 架构。
  • Codex / ChatGPT 来自官网安装器还是 Microsoft Store。
  • Codex Multi Launcher 版本和安装版/便携版。
  • 错误提示全文及 App 内复制的诊断。
  • 是否在 Parallels Desktop、VMware 等虚拟机内运行。
